# `packages/server`

This document describes how the backend package is organized and how its pieces fit together.

## High-Level Layout

```text
packages/server/
├── .env                # Runtime secrets/config (Mongo, JWT, etc.)
├── nodemon.json        # Dev reload configuration
├── package.json        # Scripts, dependencies, entry points
├── tsconfig.json       # TypeScript compiler configuration
├── src/
│   ├── index.ts        # Application bootstrap (Express, middleware, routes, DB connect)
│   ├── models/         # Type definitions for database entities
│   ├── routes/         # HTTP route handlers (controller layer)
│   └── services/       # Service layer for database interactions
├── dist/               # Compiled JS output (generated)
└── trace.log           # Runtime/debug trace output
```

## Layer Responsibilities

### 1) Bootstrap Layer (`src/index.ts`)

- Creates and configures the Express app.
- Registers middleware: static hosting, CORS, JSON body parsing.
- Mounts route groups:
	- `/api/auth` (public + token issuance)
	- `/api/users` (protected)
	- `/api/runs` (protected)
- Connects to MongoDB via `services/mongo.ts` before starting the server.

### 2) Route Layer (`src/routes/*`)
- Defines Express route handlers for each endpoint.
- Uses middleware for authentication (`authenticateUser`) on protected routes.

Routes are responsible for request validation, auth checks, HTTP status codes, and calling services.

### 3) Service Layer (`src/services/*`)

Services encapsulate database behavior and keep data logic out of route files.

### 4) Model Layer (`src/models/*`)

- Defines TypeScript interfaces for database entities (e.g., `User`, `Run`).
- Provides shared type exports (`models.ts`) for reuse across packages.

## Request Flow

Typical protected request flow:

1. Client sends request with `Authorization: Bearer <token>`.
2. `authenticateUser` verifies JWT and attaches `userId`/`userType` to `req`.
3. Route handler validates inputs and calls the relevant service.
4. Service performs Mongoose query/update.
5. Route translates result/errors into HTTP response.

## Build and Runtime Structure

- Source files: `src/`
- Build output: `dist/` (generated by `npm run build`)
- Development loop: `npm run dev` (watch + build + run)
- Production-style start: `npm run start`

## Design Summary

The package follows a clear layered backend architecture:

- **Entry/bootstrap** (`index.ts`)
- **Transport/controller** (`routes`)
- **Business/data access** (`services`)
- **Domain typing** (`models`)
